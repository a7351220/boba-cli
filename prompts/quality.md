# BOBA 日報品質自檢清單

> 主流程請參考 `SKILL.md`。

## 品質自檢清單

產出後逐項檢查：

### 來源與重複（硬性門檻，不過 = FAIL）

- [ ] 每則新聞都有 ≥1 個來源連結嗎？（沒連結 = 不能放進日報）
- [ ] 有沒有重複前兩天已報導的事件？（無新進展的重複 = 不能放）
- [ ] TG 版和 Twitter 版都準備好了嗎？（兩版同時發送，只發一次）

### 內容品質

- [ ] 有先讀前兩天日報嗎？已建立 blacklist？
- [ ] 涵蓋三大領域了嗎？（幣圈、AI、國際）
- [ ] 固定 8 則嗎？幣圈至少 4 則？每則整合 2-3 個相關事件？
- [ ] 同領域的新聞有排在一起嗎？沒有跳來跳去？
- [ ] 每則是一個**故事**嗎？有用對比、因果、趨勢串聯？
- [ ] 所有數字都是今天搜尋確認過的最新數據嗎？
- [ ] 每則有 2-3 個來源連結嗎？來源是可信的一手媒體嗎？
- [ ] 最重磅的新聞有放在前面嗎？

### 語氣與風格

- [ ] 語氣是溫和友善的嗎？有沒有不小心變得嗆辣或製造恐慌？
- [ ] 有沒有出現 BOBA 禁用語或 voice-style 禁用語？
- [ ] 有沒有給出明確投資建議？
- [ ] 壞消息有沒有包得溫暖？有沒有給方向感？
- [ ] 正負消息有平衡嗎？
- [ ] 每則都能回答「跟我的錢有什麼關係」嗎？
- [ ] 有沒有翻譯腔？大聲唸一遍，聽起來像人話嗎？
- [ ] 讀完整份日報，讀者會覺得「被照顧到了」嗎？

### 配圖與檔案路徑

- [ ] AI 底圖存到 `boba/output/drafts/daily-{MMDD}.png` 了嗎？（草稿放 drafts/）
- [ ] 完成圖（overlay 後）存到 `boba/output/daily-{MMDD}-final.png` 了嗎？

### 發送與存檔

- [ ] 已存檔到 `boba/history/{YYYY-MM-DD}.txt` 了嗎？
- [ ] TG 版和 Twitter/X 版都發送到 `@test3635` 了嗎？（同時發，只發一次）

## 自動品質檢查規則（Phase 3 用）

以下項目若 FAIL，pipeline 自動修復後重新檢查（最多重試 2 次）：

| 檢查維度 | FAIL 條件 | 自動修復方式 |
|----------|-----------|-------------|
| 來源完整性 | 任一則新聞沒有來源 URL | 移除該則，或補搜來源 |
| 重複檢查 | 與前兩天 blacklist 重複 | 移除該則，補上新主題 |
| 語氣合規 | 出現禁用語或翻譯腔 | 重寫該段落 |
| 格式正確 | 則數不足或領域缺失 | 補充新聞或調整分類 |
| 雙版本 | 缺 TG 或 Twitter 版 | 補產缺少的版本 |

> 2 次重試後仍 FAIL → 停下來告知使用者。
